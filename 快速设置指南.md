# 快速设置指南

## 🚀 5分钟快速启动

### 步骤 1：安装依赖

```bash
npm install
```

### 步骤 2：配置 Clerk（必需）

1. **注册 Clerk 账号**
   - 访问：https://clerk.com
   - 点击 "Sign Up" 注册（可以用 GitHub 账号快速注册）

2. **创建应用**
   - 登录后点击 "Create Application"
   - 应用名称：`my-blog`（随便起）
   - 选择认证方式：Email（最简单）

3. **获取 API Keys**
   - 在 Dashboard 中找到 "API Keys"
   - 复制 `Publishable Key`（以 `pk_test_` 开头）
   - 复制 `Secret Key`（以 `sk_test_` 开头）

4. **配置环境变量**
   - 复制 `.env.example` 为 `.env.local`
   - 填入 Clerk Keys：

```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_你的key
CLERK_SECRET_KEY=sk_test_你的key
```

### 步骤 3：配置 OpenAI（可选，用于 AI 功能）

1. **获取 API Key**
   - 访问：https://platform.openai.com
   - 注册/登录账号
   - 点击 "API Keys" → "Create new secret key"
   - 复制 API Key（以 `sk-` 开头）

2. **配置环境变量**
   - 在 `.env.local` 中添加：

```env
OPENAI_API_KEY=sk-你的key
```

**注意**：如果没有配置 OpenAI API Key，AI 功能会返回模拟结果，仍然可以演示功能。

### 步骤 4：启动项目

```bash
npm run dev
```

访问：http://localhost:3000

### 步骤 5：测试功能

1. **用户认证**
   - 点击右上角用户图标
   - 注册/登录账号

2. **创建文章**
   - 登录后点击"写文章"
   - 创建一篇文章

3. **AI 摘要生成**
   - 打开文章详情页
   - 点击"生成摘要"按钮
   - 查看 AI 生成的摘要

## ⚠️ 常见问题

### Q: Clerk 配置后还是无法登录？

A: 检查：
1. `.env.local` 文件是否存在
2. 环境变量名称是否正确（注意大小写）
3. 重启开发服务器（`npm run dev`）

### Q: AI 摘要生成失败？

A: 
1. 如果没有配置 `OPENAI_API_KEY`，会返回模拟结果（正常）
2. 如果配置了但失败，检查 API Key 是否正确
3. 检查 OpenAI 账号是否有余额

### Q: 创建文章时提示需要登录？

A: 这是正常的，说明权限控制正常工作。点击右上角用户图标登录即可。
