## 🎯 面试问题准备

### 可能的问题和回答

**Q: 为什么选择 SQLite 而不是 PostgreSQL？**
A: "SQLite 适合小型项目，无需额外配置，支持 Navicat 连接便于管理。如果项目规模扩大，可以轻松迁移到 PostgreSQL/Supabase。"

**Q: 后端为什么用 Node.js 而不是 Python？**
A: "我熟悉 Python（FastAPI），这个项目用 Node.js 是为了展示全栈 JavaScript 开发能力。如果需要，我可以快速用 Python 重写后端。"

**Q: AI 功能是如何实现的？**
A: "实现了完整的 RAG 工作流：1) 从数据库检索文章内容；2) 构建提示词模板；3) 调用 OpenAI API。展示了知识检索、提示词工程和 AI 调用的完整流程。"

**Q: 如何保证代码质量？**
A: "使用 TypeScript 保证类型安全，组件化设计提高可维护性，完善的错误处理和加载状态提升用户体验。代码结构清晰，AI 工具可以理解和迭代。"

**Q: 如何快速交付？**
A: "使用 Cursor 等 AI 工具辅助开发，快速生成代码、集成第三方库、处理错误。遵循 'Done is better than perfect' 的原则，先实现 MVP，再迭代优化。"
